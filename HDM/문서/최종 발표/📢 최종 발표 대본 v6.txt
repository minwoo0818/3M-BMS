2. 프로젝트 배경 
왜 탄소 배출 관리인가? 이제 탄소 배출은 기업의 생존 문제입니다.

탄소 배출은 우리가 에너지를 사용하는 모든 과정에서 발생하며, 이를 줄이지 못하면 기후 변화를 가속화하게 됩니다. 이제 ESG 경영, 탄소 중립은 기업에 있어 선택이 아닌 필수 키워드가 되었습니다.

아래는 글로벌 탄소 정책 동향입니다. 미국의 IRA(인플레이션 감축법), EU의 CBAM(탄소국경조정제도) 등 강력한 국제 규제가 시행 중이며, 우리나라도 Net-Zero 2050 탄소중립 선언과 함께 2030년 온실가스 40% 감축이라는 도전적인 목표를 세웠습니다.

특히 유럽의 CBAM 같은 경우는 특정 수입품에 대한 탄소세 부과합니다. 유럽 수출 기업이 환경 부담 비용 부담하되어 기업 입장에서 부담이 늘게됩니다.

다음으로 저희가 사용할 탄소 배출 기준에 대해서도 간단히 알아봐야겠죠? 
탄소 배출은 모두 같은게 아니라 Scope 라는게 있습니다.
국제 표준인 GHG Protocol의 기준을 따르면 SCOPE는 온실가스 배출을 발생 지점에 따라 구분한다고 합니다.

Scope 1(직접 배출): 기업이 직접 소유한 차량 운행, 시설 등에서 발생하는 배출입니다. 


Scope 2(간접 배출): 구매한 전기나 스팀, 열 사용 과정에서의 배출입니다. 


Scope 3(기타 간접 배출): 공급망, 물류, 임직원 출퇴근 등 기업 가치 사슬 전반에서의 배출을 의미합니다.


Scope에 대해 감이 오시나요?(한박자 쉬고)

저희 프로젝트에 관련된 부분은 (클릭) Scope1의 회사소유차량, (클릭) Scope3의 타 회사차량과 출퇴근 차량을 생각하시면 되겠습니다.


3. 프로젝트 설계: 요구사항 및 시스템 구조 
그렇다면 왜 이 프로젝트를 시작하게 되었을까요? 
앞서 말한 국제 정세에 따르면 저희는 감축을 해야합니다. (강조)감축을 하려면 현재 우리가 얼마나 배출하고 있는데? 어디에서 배출하고 있는데? 알아야겠죠?

그리고 만약 감축을 위한 활동을 했습니다. 그결과를 어떻게 증명 할까요? 
실생활의 예를 한가지 들어보겠습니다. 집에서 전기를 절약하는 활동을 했습니다. 얼마나 줄였는지 어떻게 알까요? 다들 전기세 고지서를 한번쯤은 보셨을 겁니다. 전년도 대비 얼마나 줄었고, 전월대비 얼마나 줄었는지 수치와 가격으로 한번에 알 수 있습니다.

하여 저희는 차량에 대해 배출량을 집계해 보기로 하였고 그것을 수치화하여 값과 추이를 한눈에 볼 수 있는 프로그램을 만들고자 하였습니다.

특히 Scope 1의 고객사 납품 차량과 Scope 3 직원들의 입출 정보를 분석하는 데 집중했습니다. 
요구사항으로는 탄소배출 데이터 자동화 관리, 협력사 관리, 대시보드, 탄소중립 목표 설정을 통해 정확한 관리 체계를 구축하고자 했습니다.

다음으로는 시스템 흐름도 입니다.

저희 시스템은 수작업을 최소화하기 위해 기존 인프라 데이터를 연동합니다. 

왼쪽 위 부터 보시면 먼저 출입 차량 입니다. 방문차량의 정보가 전달됩니다 -> 입차 정보를 관리자가 입력하면 나이스 파크 DB에 저장됩니다. 그러면 이제 차량이 출입하면 입력된 차량으로 인식하며 차량 차단기가 올라가면서 차 번호와 출입시간에 대한 입차 정보가 나이스파크 DB에 저장됩니다.

아래 줄을 보시면 직원은 출퇴근시 체크인 하는 기록이 S1 DB에 저장이 됩니다. 

이 2가지의 데이터를 저희 프로그램에 업로드 시키면 출입차량 같은 경우에는 에스원으로 계산되는 직원들을 제외한 나머지 데이터가 출퇴근 같은 경우에는 휴일, 결근 같은 출근하지 않은 데이터를 제외한 정상 데이터들이 업로드되어 저희 서버에서 관리되는 '차량 연비'/'연료'와 '이동 거리','탄소 배출계수'에 따른 탄소 배출량을 자동으로 계산해 프로젝트에서 모니터링 할 수 있게 됩니다.

메인페이지 입니다. 

먼저 관리 페이지를 보여드리겠습니다.

차량 기준정보 엑셀 관리 페이지 입니다. 저희가 관리하는 엑셀을 업로드 하면 어떤 데이터가 업로드 되는지 화면에 보이게 됩니다. 새로 업로드 하기 때문에 모두 신규로 표시되는것을 보실 수 있습니다. 엑셀 다운로드를 누르면 현재 DB에 관리되고 있는 모든 차량의 데이터가 다운로드 됩니다. 이 엑셀 파일을 수정해서 재업로드도 할 수 있습니다.

다음으로는 차량 기준정보 단일 등록 페이지 입니다. 신규 차량, 협력사, 차종, 운행 목적등을 개별로 추가등록 할 수 있습니다. 차량 번호 같은 새로운 항목들은 직접입력해야하며, 기존에 등록되어 있는 항목들은 선택시 자동 반영됩니다. 현재 화면을 보시면 협력사명을 선택하면 편도거리가 자동 반영됩니다. 운행목적도 등록되어 있는것중 선택하면 반영됩니다. 차량 등록일 같은 경우는 타고 다니던 차가 바꼈을 경우 선택해주시면되고, 차종과 연료 종료를 선택하여 등록하면 신규 차량이 등록됩니다. 

다음은 출입 데이터 업로드 입니다. 저희 프로젝트에서는 선택한 날짜와 맞지 않는 데이터를 넣으면 오류가 뜨고... (한박자 쉬고)올바른 데이터를 넣으면 업로드 됩니다. 등록을 누르면 안내 문구가 한번 더 뜨고, 다시 등록을 누르면 등록이 됩니다. .   (한박자 쉬고) 이미 등록한 데이터를 다시 등록하려고 하면 다음과 같은 안내 문구가 뜹니다. 모든차가 등록되어 들어오지 않기 때문에 등록되지 않은 차는 업로드시 빨간색으로 표시됩니다. (한박자 쉬고) 업로드한 데이터에서 차량 번호 혹은 사원번호로 검색이 가능합니다. (마지막) 등록이 완료되어도 안내문구가 뜹니다. 

다음은 목표 설정 입니다. 신규등록을 누르면 목표 설정 화면으로 전환됩니다. 등록할 목표 년도를 선택합니다. 직접 입력은 말그래도 모두 직접입력합니다. 기준실적 대비 설정은 DB에 있는 실제 배출 데이터를 가져옵니다. 그 후 비율적용을 누르면 해당 비율대로 1~12월 목표가 설정됩니다. 방금까지 Scope1이였고 Scope3도 등록해보겠습니다. Scope3는 90%로 비율적용을해서 아래 균등 분배를 해보겠습니다. 그러면 실적 데이터가 12로 나눠서 목표가 설정됩니다. (숨 쉬고) Scope1과 3 모두 설정하고 나면 각각의 합이 자동으로 총 배출량으로 집계 됩니다. 

다음은 아까 설명 드렸던 대시보드 입니다. 자세히 설명드려 잠시 보고 넘어가도록 하겠습니다.

다음은 조회페이지 입니다. 기간별 탄소 배출량 페이지 입니다. 날짜를 선택해 조회를 누르면 해당 기간에 대한 배출량, 운행거리를 전년도와 비교해 확인 할 수 있습니다.

다음은 협력사별 페이지 입니다. 해당 연도 월별 협력사 지역별 배출량과 현대정밀을 제외한 배출량 Top5를 볼 수 있습니다. 아래 데이터 테이블에 각 협력사별 배출량을 확인 및 검색 할 수 있습니다.

다음은 운행목적별 페이지 입니다. 총배출량, Scope1, Scope3, 기타로 운행 목적 마다 나눠서 확인 할 수 있습니다. 월을 전체를 선택시 파이차트와 월별 추이가 모두 보이며, 월을 선택시 파이차트만 보입니다. 월별 추이에서는 선택한 항목들만 비교해서 볼 수 있습니다. 항목이 많으면 검색도 가능합니다. 공급 유형, 공급 고객, 연료별 페이지는 같은 기능을 하기 때문에 생략 하도록 하겠습니다. 

다음은 목표 대비 탄소 배출량 페이지 입니다. 대시보드에 있던 목표를 연도별 Scope별로 확인 비교 할 수 있습니다. 

마지막으로 저감활동 관리 페이지 입니다. 활동 기간, 활동명, 내용, 금액, 기대효과, 사진을 업로드 하여 작성 할 수 있습니다. 조회페이지에서 뷰어가 조회도 할 수 있습니다.

5. 기대효과 및 향후 발전 방향 
기대효과 : 본 시스템을 통해 기업은 탄소배출량을 시각화하고 명확한 감축 목표를 설정할 수 있습니다.
또한 ESG 규제에 선제적으로 대응할 수 있습니다. 

또 향후 발전 방향으로는 AI 기반의 배출량 예측 모델을 도입하고, 타 시스템과의 연계를 위한 API 개발을 통해 Scope 1, 2, 3를 통합 관리하는 모바일 환경도 구현한다면 더 좋은 프로그램이 될 수 있을 것입니다.
